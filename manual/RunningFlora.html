Запуск Flora-2

Flora-2 полностью интегрирована в базовый движок Prolog, включая его модульную систему. В частности, модули Flora-2 могут вызывать предикаты, определенные в других модулях Prolog, а модули Prolog могут запрашивать объекты, определенные в модулях Flora-2. В настоящее время XSB является единственной платформой Prolog, на которой может работать Flora-2, поскольку она сильно зависит от таблификации и обоснованной семантики для отрицания, обе из которых доступны только в XSB.

Самый простой способ получить представление о системе - запустить оболочку Flora-2 и начать вводить запросы в интерактивном режиме. Самый простой способ сделать это - использовать сценарий оболочки 

... /flora2/runflora

Где «...» - это каталог, в который загружена Flora-2. К примеру, 

~ / ENGINEDIR / flora2 / runflora

На данный момент Flora-2 берет верх, и синтаксис F-логики становится нормой. Чтобы вернуться в командный цикл Prolog, введите Control-D (Unix) или Control-Z (Windows) или 

Flora2? - \ end.

Если вы часто используете оболочку Flora-2, есть смысл определить алиас, скажем (в Bash): 

alias ​​flora2 = '~ / ENGINEDIR / flora2 / runflora'

Затем Flora-2 можно вызвать непосредственно из командной строки, набрав flora2. Можно даже заставить Flora-2 выполнить команду при запуске. Например, набрав 

Flora2 -e "\ help."

В командном окне вашей операционной системы приведет к тому, что система выполнит команду справки сразу после инициализации. Затем отображается обычное приглашение оболочки Flora-2.

Flora-2 поставляется с несколькими демонстрациями, которые находятся в 

... / flora2 / demos /

Демо можно запустить с помощью команды demo { demo-filename }. в приглашении Flora-2, например, 

Flora2? - demo {flogic_basics}.

Нет необходимости вносить изменения в демонстрационный каталог, так как демо {...} знает, где найти эти примеры.

Файл инициализации. Когда Flora-2 запускается, он сначала выполняет команды, найденные в файле инициализации, если он указан и существует. Файл инициализации указан как значение переменной среды FLORA_RC_FILE используемой операционной системы. Если эта переменная не установлена ​​или значение этой переменной не является читаемым обычным файлом, файл инициализации игнорируется. Команды в файле инициализации могут быть любыми запросами или командами Flora-2. Они должны быть указаны точно так же, как они были бы написаны в интерактивной оболочке Flora-2, т.е. без префикса запроса «? -», и они должны быть завершены точкой. Например, если для FLORA_RC_FILE установлено значение ~ / test / myrc (примечание: суффикс .flr не требуется), и этот файл содержит 

WriteLn ( 'Добро пожаловать!')@ \ plg.
вставить {Foo (бар)}.

То будет напечатано сообщение «Добро пожаловать!» , и будет добавлен факт foo (bar).

Для файла инициализации есть одно ограничение: комментарий не может быть в конце.

Также обратите внимание, что параметр командной строки -e, упомянутый ранее, выполняется после команд файла инициализации.
